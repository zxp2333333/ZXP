Introduction Of Git

Git 是一个免费的开源的分布式版本控制系统，可以快速高效地处理各种项目

Git 易于学习，性能快，有廉价的本地库

  1. 什么是版本控制

    2. 记录文件内容变化，以便查阅版本更迭情况的系统，可以查看历史版本并进行切换

  2. 集中式版本控制

    3. 利用单一集中管理的服务器，协同工作的人通过客户端连接到服务器，最大缺点就是服务器的单点故障问题

  3. 分布式版本控制

    4. 通过远程库的克隆来基于自己的本地库来进行推送修改

  4. Git的工作机制

    5. 工作区->暂存区->本地库，提交到本地库时，会产生历史版本

Commands In Git

  1. 基本的linux命令
    1. cd: 改变目录
    2. cd .. 回退到上一个目录
    3. pwd: 显示当前目录路径
    4. ls(ll): 都是列出当前目录中的所有文件,但是ll更为详细
    5. touch: 新建一个文件
    6. rm: 删除一个文件
    7. mkdir: 新建一个文件夹
    8. rm -r: 删除一个文件夹
    9. mv 文件 目标文件夹: 移动文件
    10. reset: 初始化/清屏
    11. history: 查看历史命令
    12. help: 帮助
    13. exit: 退出
    14. #: 表示注释
  2. Git 基本命令
    1. git config -l: 查看git配置
    2. git config --global user.name + 名称: 修改名称
    3. git init: 初始化
    4. git clone + url: 克隆远程代码
    5. git status 文件: 查看文件状态,不加文件则查看所有文件状态
    6. git commit -m "new file zxp.txt": 提交到本地仓库
  3. 忽略文件
    1. # .txt: 忽略所有以txt结尾的文件
    2. !lib.txt: lib.txt为例外，不会被忽略
    3. /temp: 忽略除temp以外的其他目录
    4. build/: 忽略bulid文件夹下所有文件
    5. doc/*.txt: 忽略doc目录下.txt结尾的文件，但子目录下的不会
